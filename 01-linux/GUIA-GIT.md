# Guia de Comandos Git Essenciais

1. **git init**
   * *O que faz:* Inicia um repositório novo na pasta.
   * *Quando usar:* Apenas na criação do projeto.

2. **git status**
   * *O que faz:* Mostra o estado dos arquivos (modificados, novos, etc).
   * *Quando usar:* O tempo todo, para não se perder.

3. **git add .**
   * *O que faz:* Prepara todos os arquivos modificados para o próximo commit.
   * *Quando usar:* Quando terminou uma etapa do trabalho.

   4. **git commit -m "mensagem"**
   * *O que faz:* Salva a versão atual no histórico.
   * *Quando usar:* Para fechar um pacote de alterações.

5. **git log**
   * *O que faz:* Mostra a lista histórica de commits.
   * *Quando usar:* Para ver o que foi feito anteriormente.

6. **git branch [nome]**
   * *O que faz:* Cria uma ramificação paralela.
   * *Quando usar:* Para testar algo sem estragar o código principal.

7. **git checkout [nome]**
   * *O que faz:* Troca de branch.
   * *Quando usar:* Para navegar entre versões ou branches.

8. **git merge [nome]**
   * *O que faz:* Funde uma branch na atual.
   * *Quando usar:* Para trazer as novidades da branch de teste para a principal.

9. **git push**
   * *O que faz:* Envia as alterações para o GitHub (nuvem).
   * *Quando usar:* Para fazer backup ou entregar o código.

10. **git clone [link]**
    * *O que faz:* Baixa um repositório da nuvem para o PC.
    * *Quando usar:* Quando formatar o PC ou trocar de máquina.
